## 🔍 详细分析：剑魂百分比综合弹性偏差

### 问题描述

v1735验证脚本（`verify_v1735.py`）对剑魂百分比综合的计算采用了**四区乘法模型**：

```python
# 脚本中的计算公式（4区）
swordman_pct_calc = (1 + 38.13%) * (1 + 5.21%) * (1 + 7.97%) * (1 + 1.20%) - 1
# 结果: 58.79%
```

而标准值45.70%所对应的公式为**三区乘法模型**（不含独立攻击）：

```
标准值计算: (1 + 38.13%) * (1 + 5.21%) * (1 + 1.20%) - 1 = 47.07%
```

### 独立验算结果

| 方法 | 乘区组成 | 计算结果 | 与标准值45.70%差异 |
|------|----------|----------|-------------------|
| 方法A（脚本当前） | STR+PHY+INDP+Crit（4区） | **58.79%** | **+13.09pp** |
| 方法B（不含独立攻击） | STR+PHY+Crit（3区） | **47.07%** | **+1.37pp** |
| 标准值 | — | **45.70%** | — |

### 结构性问题

**关键矛盾**：狂战士百分比综合计算公式为**三区模型**（STR+PHY+Crit），**不包含独立攻击**。这是正确的，因为根据 `BERSERKER.md` 知识库明确标注：

> "固伤不受力量/物理攻击；百分比受力量/物理攻击/武器强化"

这意味着百分比技能的加成乘区不包含独立攻击。然而，剑魂百分比综合的计算却**包含独立攻击**，与狂战士的计算方法不一致。

**进一步分析**：方法B（排除独立攻击后）得到47.07%，仍与标准值45.70%存在1.37pp差异。此差异可能源于：
1. 剑魂基础力量值813.0可能存在轻微偏差
2. 游戏机制中存在百分比乘区递减/修正机制（FAAL弹性偏差体系已确认）

### 弹性偏差体系评估

该差异已被纳入FAAL弹性偏差体系（v1701发现、v1731确认固化）。FAAL弹性偏差体系记录显示"3项偏差确认"，本次剑魂百分比综合偏差为其中一项。

**审计意见**：弹性偏差体系的引入是合理的——当理论公式与经验标准值存在结构性差异时，以经验标准值为锚点、记录偏差成因，比强制修改标准值更科学。但该偏差的**根本成因**（剑魂百分比是否受独立攻击影响）仍需外部数据源确认。
